# 任务看板 (Task Board) — active-tasks.md 模板

> 复制此结构到 `{project_root}/docs/tasks/active-tasks.md`，清空示例数据后使用。
> 这是 agate 的任务汇总视图，与各任务目录下的 `.state.yaml` 配合：
> `.state.yaml` 是单任务的权威状态，`active-tasks.md` 是全局只读汇总（由主 Agent 扫描重建，不由 subagent 直接改）。

---

```markdown
# 任务看板 (Task Board)

> {项目名} 任务管理主文件
> **核心原则**：所有任务必须走 agate 流程，每个阶段有文件产出
> **位置**：`docs/tasks/` 目录下每个任务一个子目录

---

## 任务列表

### 进行中的任务

| 编号 | 任务名称 | 状态 | 阶段 | 优先级 | 依赖 | 创建日期 | 更新日期 |
|------|----------|------|------|--------|------|----------|----------|
| T001 | example-task | 🔄 进行中 | P2 | P0 | — | YYYY-MM-DD | YYYY-MM-DD |

### 待开始

| 编号 | 任务名称 | 状态 | 阶段 | 优先级 | 依赖 | 创建日期 | 更新日期 |
|------|----------|------|------|--------|------|----------|----------|
| T002 | next-task | ⬜ 待开始 | — | P1 | T001 | YYYY-MM-DD | YYYY-MM-DD |

### 已完成

| 编号 | 任务名称 | 状态 | 最终阶段 | 优先级 | 完成日期 |
|------|----------|------|----------|--------|----------|
| T000 | done-example | ✅✅ 已完成 | P8 | P0 | YYYY-MM-DD |

> **依赖列说明**：主 Agent 启动任务前检查依赖列。所有依赖任务状态为 ✅✅ 已完成 才启动。
> **active-tasks.md 是汇总视图**：不由 subagent 直接修改，由主 Agent 维护。日常更新只重写自己任务那一行（从 `.state.yaml` 派生），定期或怀疑不一致时从所有 `.state.yaml` 全表重建。
```

## 字段说明

| 字段 | 取值 | 说明 |
|------|------|------|
| 编号 | T001, T002... | 任务唯一编号，递增不重复，**不回收已取消任务的编号** |
| 任务名称 | kebab-case 短描述 | 同时作为任务目录名后缀：`docs/tasks/T001-{任务名称}/` |
| 状态 | ⬜ 待开始 / 🔄 进行中 / ⏸️ 等待 / ✅✅ 已完成 / ❌ 已取消 / ⏸️ HANDOVER | 见下方状态机说明 |
| 阶段 | P0-P8 / DONE / READY | 当前所在阶段，与 `.state.yaml` 保持一致 |
| 优先级 | P0（最高）-P3（最低） | 项目自定义优先级体系，P0/P1/P2/P3 仅为示例 |
| 依赖 | 任务编号或 — | 依赖任务必须 ✅✅ 已完成才能启动本任务 |

## 状态机简述

```
⬜ 待开始 → 🔄 进行中 → ✅✅ 已完成
                ↓ (gate 失败超限)
            ⏸️ PAUSED（见 dispatch-protocol.md L2 上溯）
                ↓ (环境受限，交接其他 Agent)
            ⏸️ HANDOVER（见 HANDOVER.md）
```

任意阶段可能 ❌ 已取消（需求变更/不再需要），取消的任务移入"已完成"表格，状态标 ❌ 已取消，编号不回收。

## 维护规则

1. **只有主 Agent 改这个文件**，subagent 不直接写 active-tasks.md（避免多 Agent 并发冲突）
2. 每次阶段推进后，主 Agent 同步更新对应任务的行（状态/阶段/更新日期）
3. 定期（或怀疑不一致时）用各任务的 `.state.yaml` 重建此文件，`.state.yaml` 是权威来源
4. 新任务编号 = 当前最大编号 + 1，不复用已取消任务的编号
