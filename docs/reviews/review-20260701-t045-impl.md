## 二、发现的问题

### 无 Critical / Important 问题

### Minor: dispatch-protocol.md:575 仍写"查询类 BDD 可不截图（断言值是唯一证据）"

这句话本身没错——查询类可不截图，但必须有断言记录文件。但现在 hook 已经强制所有 PASS 有文件引用，dispatch-protocol.md 这句话的措辞可能被误读为"查询类可以没有任何证据文件"。

**建议**：补一句"但须有断言记录文件（response.json / assert.log 等），hook 强制"。不阻塞，下一轮文档同步时处理。

### Minor: verifier.md 的 PENDING_RUNTIME_VERIFICATION 标注

verifier.md 加了"如果你产出了 Playwright 验证脚本但没有实跑，必须在 acceptance.md 显式标注 `⚠️ PENDING_RUNTIME_VERIFICATION`"。但评审 v5 否决了 PENDING 状态（与二值规则冲突 + 可被 subagent 绕过）。

**评估**：这里的 PENDING_RUNTIME_VERIFICATION 不是 acceptance 模板的状态字段，是 verifier 在正文里的标注——不与二值规则冲突（PASS/FAIL 仍然是结论，PENDING 只是"附注"）。但措辞可能被误读为"可以标 PENDING 而不标 PASS/FAIL"。

**建议**：改为"在 acceptance.md 正文标注 `⚠️ 脚本未实跑，需主 Agent 验证`"，避免 PENDING 一词。不阻塞。
