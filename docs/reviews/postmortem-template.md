## 机制触发核对清单（必填）

对照本任务中 agate 的 5 个核心机制，逐条检查：

| 机制 | 应该触发？ | 实际触发？ | 未触发后果 | 原因 |
|------|-----------|-----------|-----------|------|
| retry 记录 | 是/否 | ✅/❌ | | |
| PAUSED | 是/否 | ✅/❌ | | |
| PROD_TOUCHED | 是/否 | ✅/❌ | | |
| SCOPE+ | 是/否 | ✅/❌ | | |
| gate 验证 | 是/否 | ✅/❌ | | |

- "应该触发" = 本任务中是否有该机制的触发条件出现
- "实际触发" = 主 Agent 是否按协议执行了该机制
- 未触发 = 执行错误（不是机制缺口）
- 机制缺口 = 机制本身没有覆盖这个场景（需在下方"机制改进建议"中说明）

—— T019 教训：复盘完全漏掉 SCOPE+ 遗漏（方案从 frontend 扩为 backend+frontend 应触发 SCOPE+ 但未触发），因为复盘没有系统性核对每个机制。本清单防止此类遗漏。
