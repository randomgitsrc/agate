### 3. [NEEDS_FIX] M5 regex 末尾换行 bug

**位置**：`agate/scripts/check-gate.sh:151`

**当前 regex**：
```python
r'^gate_commands:[ \t]*\n((?:  .*\n|\s*\n)*)'
```

**问题**：`  .*\n` 要求每行以 `\n` 结尾。当 gate_commands 块位于文件末尾且无尾随换行时，最后一个键丢失。

实测：
```python
content = 'gate_commands:\n  P5: "pytest"\n  P5_e2e: "playwright"'  # 无尾随 \n
# → 只捕获 '  P5: "pytest"\n'，P5_e2e 丢失
# → P5 count = 1（应为 2），WARNING 漏报
```

**影响**：仅影响 P5 多命令 WARNING 漏报，不导致错误拦截。P2-design.md 通常有尾随换行，实际触发概率低。

**修复**：在 `content = f.read()` 后追加 `if not content.endswith('\n'): content += '\n'`

**评估**：合理。版本号跟随 git tag，不跟随 commit。与 v0.17-0.19 历次处理一致。

### C. M5 regex `  .+$` → `  .*\n`

**Plan 说**：用 `  .+$`（配合 re.MULTILINE）
**实际实施**：用 `  .*\n`

**评估**：合理且必要。Plan 的 `  .+$` 在 MULTILINE 模式下 `$` 匹配行尾但不消费换行符，导致捕获组只含第一行。改用 `  .*\n` 消费换行符修复了此问题。但引入了末尾换行边界 bug（见上方第 3 项）。
