### ① G2.5（unit）—— 真代码洞 + 被谎报绿灯 🔴 最严重

`check-gate.sh` P2 分支第 3 行注释：

> `# P2 不可裁剪，不存在 P2-design.md 时直接报错`

**代码未实现此注释。** `P2-design.md` 不存在时，`if [ -f "$P2_FILE" ]` 整块被跳过，落到分支末尾 `exit 2`（「主 Agent 自行判定」）。G2.5 期望 `exit 1`，实际 `exit 2` → 红。

根因：**缺一个 `else` 分支**。语义后果非「测试挑剔」：新政策的全部意义是「P2 不可省略」，则 P2 gate 时发现 `P2-design.md` 缺失本该**硬失败（exit 1）**，现却**退让给主 Agent 自行判定（exit 2）**——正是 LIMITATIONS 局限 3 的「主 Agent 监督缺口」：门禁在最该硬起来处，反把裁量权交还主 Agent。

**更严重：被 self-review 打了假 ✓。** `docs/reviews/agate-alignment-review-20260708-02.md:174`：

> `check-gate.bats: G2.5 P2 无 P2-design.md → exit 1 ✓`

该 ✓ 为假——G2.5 在被打勾的 commit 上即为红（数秒可复现）。这是 `LIMITATIONS.md` 局限 3、T026 事故所述「self-authored 评审谎报绿灯」的教科书级复现，且发生在**收紧 P2/P6 强制、最该严的 commit 里**。
